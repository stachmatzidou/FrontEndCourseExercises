Data Types in JavaScript
Primitive Data
Number JavaScript has only one type of numbers, which can be both integers and floating point numbers, as large as 253-1. Learn more here.
BigInt A special numeric type that provides support for integers of arbitrary length. It provides a way to represent whole numbers larger than the largest number JavaScript can reliably represent with the Number primitive. Learn more here.
String It stores a series of characters, used to represent text, inside double or single quotes. Learn more here and here.
Boolean A logical data type that represents one of two values: true or false, named after English mathematician George Boole, who pioneered the field of mathematical logic. Learn more here.
Undefined A primitive value assigned to variables that have been declared, but have not been assigned a value. Learn more here.
Null It is supposed to be something that doesn't exist. It expresses a lack of identification, indicating that a variable points to no object. Learn more here and check the difference between null and undefined here.
Symbol It is used to create unique identifiers for objects. Just like it's primitive counterparts (Number, String, Boolean), they can be created using the factory function Symbol() which returns a Symbol. Learn more here.
Complex Data
Objects They are used to store keyed collections of various data and more complex entities. In JavaScript, objects are standalone entities, with properties and type, which penetrate almost every aspect of the language. Learn more here and here.
